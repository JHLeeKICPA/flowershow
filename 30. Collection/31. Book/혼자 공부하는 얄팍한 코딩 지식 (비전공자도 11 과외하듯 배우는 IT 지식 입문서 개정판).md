## Description
혼자 해도 충분하다!
비전공자도 1:1 과외하듯 배우는 IT 지식 입문서

『혼자 공부하는 얄팍한 코딩 지식(개정판)』은 IT 지식을 처음 배우기 시작한 입문자가 ‘꼭 필요한 내용을 제대로 학습’할 수 있도록 구성했습니다. 코딩을 접해 보지 못한 사람도 부담 없이 읽을 수 있도록 전문 용어의 사용을 줄이고 일상에서 쓰는 언어로 친절하게 설명하고 있습니다. 

【개정판에서 추가된 내용】
챗GPT와 같은 생성형 AI와 최신 AI 기술을 입문자의 눈높이에 맞춰 재미있는 일러스트와 다양한 예시를 통해 소개하고 있습니다. 본 도서를 통해 IT 지식과 용어, 최신 IT 트렌드까지 한 번에 학습할 수 있습니다. 

개발이 뭔지 궁금했다면, 개발자와의 소통에서 어려움을 느꼈다면, 개발자가 되고 싶었다면! 지금 바로 IT 지식과 개념을 단단히 할 수 있는 『혼자 공부하는 얄팍한 코딩 지식(개정판)』을 펼쳐 보세요.

## 내용 정리

### 1장. 개발자 이해하기

#### 개발자란?
- 웹 개발자
	- 웹 퍼블리셔/UI 개발자
	- 프론트엔드 개발자
	- 백엔드 개발자(서버 개발자)
	- 풀스택 개발자
- 모바일 개발자(안드로이드/iOS)
- 게임 개발자
- 응용 소프트웨어 개발자
- 임베디드 개발자
- 정보보안 전문가
- 인공지능 개발자
- 데이터베이스 개발자
- 클라우드 개발자
- 데브옵스 개발자

#### 코딩, 프로그래밍, 앱, 프로그램
- 프로그래밍 > 코딩
	- 프로그래밍: 컴퓨터가 할 일의 절차와 알고리즘을 설계하는 것
	- 코딩: 프로그래밍 언어로 된 코드를 입력하는 작업
- 개발
	- 소프트웨어를 설계, 구현, 운영, 관리하는 전반적인 과정
- 소프트웨어 > 프로그램 > 시스템 프로그램, 응용 프로그램
	- 소프트웨어: 컴퓨터의 하드웨어 외 보이지 않는 부분을 통칭
	- 프로그램: 사용자의 명령에 따라 목적에 맞는 작업을 수행하는 일련의 명령 모음

#### 프로그래밍 언어
- 컴파일 언어 → *번역*
	- 프로그래밍을 완료한 후 코드 전체를 미리 기계어 등으로 컴파일한 후 실행
	- 개발시간은 늘어나지만 오류로부터 비교적 안전하고 프로그램 실행 속도 빠름
	- ex) C, C++, C#, Java, Typescript, Kotlin, Swift 등 
- 인터프리터 언어 → *통역*
	- 사전 번역 과정 없이 실행시 바로 한 줄씩 통역되어 동작
	- 신속하고 유연한 개발 과정이 필요한 소프트웨어에 많이 사용
	- ex) Javascript, Python, PHP, R 등
- IDE
	- 이클립스 → 주로 자바에 사용
	- 인텔리제이
	- 파이참 → 파이썬에 최적화
	- 안드로이드 스튜디오 → 자바나 코틀린
	- 엑스코드 → 스위프트
	- Visual Studio

#### 개발자 용어
- 디버깅
	- 프로그램 개발 중 발생하는 오류의 원인을 찾아 제거
	- 작성된 코드가 의도대로 작동하는지 수시로 확인해보는 것으로 의미를 확장해서 사용
- 빌드
	- 소스 코드 등을 모아 파일 또는 파일 묶음으로 만드는 과정
		- 빌드 번호: 출시되기 전의 프로그램 버전
	- ex) 윈도우 프로그램은 exe파일로, 모바일 앱은 apk로 변환
- 릴리즈
	- 빌드를 마친 파일을 사용자가 접근할 수 있는 환경에 배포하는 것
- 유지 보수
	- 프로그램 릴리즈 후 이어지는 버그 대응 및 기능 업데이트 등
- 리팩토링
	- 코드를 개선하여 읽기 쉽고 효율적인 코드로 만드는 작업
- 코드 리뷰
	- 개발자가 각자 맡은 부분 프로그래밍 해서 소스 코드에 반영할 때 동료 코드를 확인하고 의견 교환
- 문서화
	- 코드를 전반적으로 요약해서 설명하고 각 기능을 어느 코드에서 사용하면 되는지 등을 정리한 문서

### 2장. 인공지능 기술

### 3장. IT 업계 용어

#### 서버, 클라이언트
- 서버 호스팅(Server Hosting)
	- 서버용 컴퓨터를 임대해 주는 서비스
- 온프레미스(On-Premise)
	- 회사에서 자체적으로 서버를 갖추고 관리하는 것
- 클라우드 컴퓨팅
	- 서버, 저장소, 데이터베이스, 네트워킹, 소프트웨어 등을 클라우드로 제공
	- 비용이 높지만 쉽고 안정적으로 서버 운영 가능
	- 클라우드 종류
		- IasS
			- 서버용 인프라만 대여
			- ex) AWS의 EC2
		- PaaS
			- 서버, 운영체제, 개발환경 등을 자동으로 설치하고 제공
			- ex) AWS의 일래스틱 빈스토크, 헤로쿠
		- SaaS
			- 소프트웨어가 이미 완성된 형태로 제공
			- ex) 구글드라이브, 원드라이브 등

#### 웹사이트
- 웹 브라우저
	- html이라는 형식의 문서를 읽는데 사용되는 것
- 웹 표준
	- 브라우저마다 HTML, CSS, 자바스크립트 코드 실행 방식을 동일하게 하는 것
	- 인터넷 익스플로러는 이를 지키지 않아서 현재 대체됨
- 모바일/PC 웹 종류
	- 반응형 웹
		- 모바일, PC에 모두 자동으로 조정되도록 만드는 것
	- 적응형 웹
		- 모바일용, PC용 따로 만드는 것

#### 쿠키, 토큰, 캐시
- 쿠키
	- 브라우저에 저장되는 작은 텍스트 조각
- 세션
	- 로그인 여부 등 사용자와 서버의 관계가 기억되어 보존하는 것
	- 한 번 로그인 하면 유효기간이 끝날 때 까지 입력하지 않아도 되도록 인증
	- 장점: 사용자의 상태를 원하는 대로 통제 가능
	- 단점: 동시 접속자가 많아지면 메모리 공간이 부족
- 토큰
	- 해당 서버만이 만들 수 있는 토큰을 발급하여 상태를 저장하지 않고 로그인 여부 파악
	- 장점: 상태를 메모리에 저장할 필요 없음
	- 단점: 한 번 로그인한 사용자의 상태는 토큰 만료 전까지 제어 불가
- 캐시
	- 같은 사이트를 다시 방문하거나 동영상을 다시 시청할 때 캐시 형태로 저장 → 전송 재요청 불필요
- CDN
	- 지리적으로 분산된 여러 서버를 이용해 웹 콘텐츠를 사용자와 가까운 서버에서 전송

#### 스마트폰 앱, 웹사이트
- 네이티브 앱
	- 각 운영체제에 맞는 앱을 개발하는 것
- 크로스 플랫폼
	- 안드로이드, iOS 양쪽에서 모두 작동할 수 있는 앱을 만드는 개발도구
	- 성능이 어느정도 제한적
	- ex) 플러터, 리액트 네이티브, 닷넷 마우이
- 웹 앱
	- 모바일 앱과 비슷한 UI/UX를 지원하는 웹사이트
	- 비교적 간단하게 만들 수 있지만, 기능이 한정적
- PWA(프로그레시브 웹 앱)
	- 아이콘 추가 등 진보된 브라우저 기능을 활용해서 네이티브 앱 같은 다양한 편의기능 제공하는 웹 앱

#### 프론트엔드, 백엔드, 프레임워크 등
- 프론트엔드
	- 브라우저에서 동작하는 쪽
	- HTML, CSS, 자바스크립트, 타입스크립트
- 백엔드
	- 서버에서 작동하는 쪽
	- 자바, 파이썬, C#, PHP, 루비
- 프레임워크
	- 개발에 필요한 필수 요소와 기본적인 기능이 이미 프로그래밍 되어 잇는 제작 키트
	- 프레임워크를 기반으로 그 위에 프로그램을 만듦
	- 백엔드
		- 스프링(JAVA), 장고(Python), 플라스크(Python), 익스프레스, 닷넷 등
	- 프론트엔드
		- 앵귤러, 뷰, 리액트
- 라이브러리
	- 여러 프로그램에 들어갈 수 있는 부품
	- 프로그램을 만들기 위해 가져다 쓰는 것
- API
	- 프로그램 간 소통을 위해 만들어진 신호 체계
	- REST API
		- 보편화된 API 규칙
	- 한 주체가 다수에게 특정 기능을 개방할 때도 사용
- XML, JSON, YAML, AJAX
	- XML
		- 마크업 언어(HTML과 유사)
		- HTML은 웹 페이지 구조 개발에 사용, XML은 플랫폼 간 데이터 송수신에 사용
	- JSON
		- 대괄호, 중괄호 등으로 작성
		- XML보다 구문이 간단하고 가독성이 높아서 더 많이 사용
	- YAML
		- 하이픈을 이용해 배열 표시
		- 주로 프로그램 설정 파일과 같이 개발자가 편리하게 읽고 작성하기 위한 용도
	- AJAX
		- 자바스크립트를 이용해 서버와 브라우저가 데이터를 교환할 수 있는 통신 기능 수행

#### IP, HTTPS
- IP주소
	- 인터넷에 연결된 PC나 스마트기기를 식별할 수 있는 특수한 번호
	- 기기 자체의 주소가 아니라 기기와 연결된 네트워크 끝 단의 주소
	- 방식
		- IPv4
			- 123.123.123.123
		- IPv6
			- 1234:5678:9ABC:DEF0:1234:5678:9ABC:DEF0
	- 종류
		- 공인 IP 주소
			- 고유의 IP주소 → 웹사이트 같은 서버를 찾기 위한 용도
		- 사설 IP 주소
			- 공유기로 하나의 공인 IP 주소를 여러 기기가 함께 사용할 때 구분할 수 있는 주소 할당
			- 로컬 IP, 가상 IP로도 부름
		- 고정 IP 주소
			- 영구적으로 할당되어 변경되지 않는 주소 → 웹사이트, API, 서버
		- 유동 IP 주소
			- 남아 있는 IP 주소를 그때 그때 기기에 할당 → 일반적인 클라이언트
- 도메인, URL
	- IP주소를 이름처럼 사용할 수 있는 것
- DNS
	- 도메인을 IP 주소로 변환해서 경로를 안내
- WWW, HTTP, HTTPS
	- WWW
		- 호스트명 → 월드와이드웹
		- 하이퍼 텍스트 지원
	- HTTP
		- 통신규약을 나타냄
		- 클라이언트의 요청과 서버의 응답으로 구성된 방식
	- HTTPS
		- HTTP의 보안 취약점을 보완하기 위해 만들어진 것
		- 웹사이트의 신뢰성을 CA로부터 인증서 발급 받아서 인증
		- 검색 엔진 최적화에도 유리
- 요청 메소드와 응답 코드
	- 메소드
		- GET
			- 서버에 데이터를 요청
		- POST
			- 서버에 새로 저장될 자료를 송신(ex. 새 게시물 작성)
		- PUT
			- 특정 데이터 전체를 새로 저장(ex. 게시물 수정)
		- PATCH
			- 특정 데이터를 일부만 수정(ex. 게시물의 조회수나 좋아요 올림)
	- 응답 코드
		- 1XX
			- 요청을 성공적으로 받았으며 서버가 해당 작업을 진행 중
		- 2XX
			- 요청을 성공적으로 받았으며 요청이 이루어짐
		- 3XX
			- 요청을 수행하기 위해 다른 주소로 redirection
		- 4XX → 사용자 측 오류
			- 클라이언트 요청에 문제가 있어 수행 불가
			- ex. 404: Nof Found - 요청에 해당하는 데이터가 없음 또는 URL이 잘못 됨
		- 5XX → 서버 측 오류류
			- 요청에는 문제가 없지만 서버에 이상이 있어 응답 불가

#### 데이터와 데이터베이스

	- 오라클

		- 그래프 데이터베이스

#### 운영체제
- 커널
	- 운영체제의 핵심
	- 하드웨어와 응용프로그램 사이에서 작동하며 모든 일을 통제
- 사용자 인터페이스
	- GUI(Graphic User Interface)
		- 그래픽 사용자 인터페이스
	- CLI(Command Line Interpreter)
		- 명령줄에 텍스트를 입력함으로써 컴퓨터에 일을 시키는 환경
		- 장점
			- 컴퓨터 자원을 덜 소모
			- 반복되는 작업을 일정하게 진행하는 데는 더 간편하고 정확
			- 네트워크 환경에 더 적합

### 4. 프로그래밍 용어
#### 컴퓨터 작동방식
- CPU
	- 코어
		- 손이 몇개인지
	- 스레드
		- 코어 1개가 몇개처럼 일하는지
	- 클럭
		- 코어가 얼마나 빠르게 일하는지
- 메모리
	- 캐시
		- CPU에 내장된 형태로 가장 빠르게 접근 가능
	- 주 기억장치, 램(RAM)
	- 보조 기억장치
- 입출력 장치(I/O)
- 프로세스, 스레드
	- 프로세스
		- 프로그램이 메인 메모리로 올라와 실행되고 있는 상태
		- 프로세스 처리 방법
			- 병렬 처리
				- 2개 이상의 코어가 각기 다른 프로세스를 실행
			- 병행 처리
				- 하나의 코어가 여러 프로세스를 돌아가면서 조금씩 처리
			- 멀티 프로세싱
				- 여러 개의 프로세스를 병렬, 병행으로 함께 진행
	- 스레드
		- 한 프로세스 안에 하나 이상 진행될 수 있는 일의 단위
		- 여러 스레드가 함께 실행되는 것을 멀티 스레딩이라고 함

#### 바이트, 자료형, 인코딩
- 비트
	- 0과 1 두 값을 가지는 것으로 컴퓨터가 다루는 데이터의 최소 단위
- 바이트
	- 비트가 8개 모인 것
- 자료형
	- 프로그래밍 언어에서 여러 종류의 데이터를 저장하는 방식
- 문자 인코딩
	- 사람이 사용하는 문자를 컴퓨터가 인지 가능한 숫자로 변환
	- 유니코드
		- 전 세계에서 사용되는 대부분의 문자를 포함한 문자열 set

#### 프로그래밍 언어 분류
- 절차지향 프로그래밍
	- 소스 코드를 위에서부터 차례대로 읽음
	- 장점
		- 실행속도가 빠름
	- 단점
		- 모든 구성요소가 유기적으로 연결되어 사소한 문제만 있어도 시스템 전체가 오류
		- 실행 순서가 정해져 있기 때문에 소스 코드의 순서가 바뀌면 결과가 바뀜
- 객체지향 프로그래밍
	- 프로그램을 다수의 객체로 만들고 이 객체끼리 서로 상호작용 하는 방법
	- 장점
		- 유지 보수가 쉽고 생산성이 높음
	- 단점
		- 모든 객체의 역할과 기능을 이해해야 하기 때문에 설계에 시간이 오래 걸림
		- 처리속도가 느림
		- 대체로 난이도가 높음
- 함수형 프로그래밍
	- 함수는 특정한 기능을 하는 소스 코드를 따로 빼서 묶어놓은 것
	- 변수 사용을 최소화 함으로써 복잡한 코드의 오류를 줄임
- 매니지드 언어와 언매니지드 언어
	- 메모리를 프로그래밍 언어가 직접 관리하면 매니지드 언어, 개발자가 직접 관리하면 언매니지드 언어
	- C, C++이 몇 안되는 언매니지드 언어

#### 오류
- 컴파일 오류
	- 소스 코드에 오타나 문법적 결함 등으로 컴파일 과정에서 오류 발생
- 런타임 오류
	- 프로그램 실행 중 발생하는 오류
	- 오류 예시
		- Null에 의한 오류
		- 0으로 나눠서 생기는 오류
		- 의도하지 않은 무한 루프
- 논리 오류
	- 컴파일이나 실행 자체는 성공하지만 잘못된 결과를 반환하는 오류
- 예외 처리
	- 오류 상황에도 프로그램을 끄거나 리셋하지 않고 처리하는 방법
	- try-catch / try-escept / rescue 문 등 사용
- 메모리 누수, 가비지 컬렉션
	- 메모리 누수
		- 메모리 공간에 불필요한 데이터가 쌓여서 남아 있는 것
	- 가비지 컬렉션
		- 메모리 관리를 상당 부분 자동으로 처리해 주는 기능

#### 개발에 필요한 여러가지 도구
- 패키지 매니저
	- 패키지를 편리하고 안전하게 다루기 위한 도구
	- 종류
		- 운영체제 패키지 매니저
			- ex. 리눅스 - apt-get, yum, pacman
		- 프로그래밍 언어 패키지 매니저
			- ex. 파이썬-pip / 자바-Marven, Gradle / 자바스크립트 - NPM, Yarn 등
- 환경 변수
	- 운영체제 입장에서 프로세스를 실행시키기 위해 참조하는 값을 담은 변수
	- 용도
		- 필요한 프로그램의 실행 경로
		- 개발 환경 용도 구분
		- 민감한 요소의 보안
- 런타임 환경
	- 프로그래밍 언어가 컴퓨터에서 일을 할 수 있도록 해주는 요소
	- 프로그래밍 언어 이름 뒤에 런타임이 붙으면 이를 지칭
	- ex. JRE, Node 등
- SDK
	- 소프트웨어를 개발하는데 사용되는 키트
	- ex. JDK, .Net SDK
- 라이브러리, 프레임워크
